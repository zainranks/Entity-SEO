# QA checklist

## Entity quality
- [ ] Primary entity is explicit and unique
- [ ] Aliases do not merge distinct entities
- [ ] Every important relationship has evidence
- [ ] Confidence threshold is respected

## Page architecture
- [ ] Page has a distinct intent and task
- [ ] No unresolved cannibalization
- [ ] Page depth justifies a URL
- [ ] Parent, child, and sibling relationships are coherent

## Content
- [ ] Brief answers the user task
- [ ] No entity stuffing
- [ ] No fabricated facts or citations
- [ ] Regulated claims are flagged for review
- [ ] Differentiation is specific

## Technical
- [ ] Outputs validate against schemas
- [ ] Internal links use valid targets
- [ ] Schema reflects visible content
- [ ] `sameAs` only represents identical entities
